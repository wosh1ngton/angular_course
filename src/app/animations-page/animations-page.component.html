Trabalhando com animações

Há duas formas principais de trabalho com animações, através do CSS e através do Javascript. o css pode ser utilizado para animar e transicionar elementos, além das instruções que possui, há libraries css que facilitam o processo de animação, por exemplo, a library animate.css. Para utilizar a library é bem simples, basta baixar a library e incluir o link de referência no cabeçalho da sua página, ou se preferir, utilizar o CDN. A outra abordagem de animação, JS, possui como vantagem a maior abrangência e controle da animação. Assim como no caso do css, há diversas libraries que podem ser utilizadas para animar a sua página, destacam-se: jQuery, GSAP, Zepto, Web Animations API. Esta última é uma especificação que é implementada no Angular pelo módulo @angular/animations. Este módulo possui uma série de funções para criar animações, tais como: trigger(), transition(), state(), animate() e outras.

Reutilizando o trigger de animação

Para fazer reuso da animação crie um novo arquivo, animations.ts, e cole a função trigger no arquivo exportando-o com o nome que irá usar para a chamada e importando os módulos necessários, por último, chame pela função criada no arquivo na array animations do seu componente.

Keyframes

Keyframes são utilizados para animações mais complexas. é um parâmetro da função animate, a qual recebe um valor para a transição e depois uma array de keyframes, essa array pode conter vários styles.

A maior parte dos navegadores possui suporte a animação, para os casos de exceção cabe o uso do polyfills.ts, que faz exatamente esse trabalho de adequação de códigos mais modernos de js para que possam ser executados em navegadores antigos. Há diversas outras funcionalidades do pacote de @angular/animations, como query, group, animateChild, stagger. Todas essas funcionalidades podem ser melhor estudadas na documentação, o objetivo deste texto era conferir uma visão geral das possiblidades.

Custom states

Caso haja necessidade de animar elementos que estão sempre no DOM e portanto não mudam de estado, você precisará criar estados customizados e trabalhar com eles. pra isso existe a função state.

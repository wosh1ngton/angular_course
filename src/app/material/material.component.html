Organizando seu código

A fim de manter sua aplicação coesa e organizada, uma boa prática é criar um módulo específico para os módulos do material, observe que o app.module cresce consideravelmente a medida que mais componentes do material são acrescidos, acaba poluindo o arquivo e isso não é interessante para a manutenção do código. Para resolver essa questão, crie um novo módulo utilizando o angular CLI e transfira os módulos da array de imports para a array de exports do novo módulo criado, na array de imports do app.module você chamará apenas um módulo que contém as referências para os componentes do Material, aqui chamei de MatComponentsModule.

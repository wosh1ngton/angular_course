Visão Geral

Um Router é um serviço que provê navegação entre views e manipulação de URLs

3 passos para configurar a navegação

1º Configurar as rotas: é o mapeamento de um caminho - path - para um componente;
2º Adicionar um router outlet: é onde é mostrado o componente correspondente quando uma dada rota se torna ativa;
3º Adicionar links: pode ser feito utilizando o atributo routerLink nos elementos de âncora.

Uso de caminhos relativos

A barra "/" indica que o caminho é absoluto e deve iniciar a partir da string, logo após a barra, por exemplo, se no routerLink o valor indicado é
routerLink='/followers'
o caminho a ser buscado será host:porta/followers
Caso o valor indicado no routerLink seja apenas a "/"
routerLink='/'
a engine do Angular buscará por host:porta
Uma outra possibilidade é informar apenas a string, nesse caso, a engine utilizará o caminho existente na barra de endereços + o valor informado no routerLink, por exemplo, se na barra de endereços o valor é host:porta/followers e o valor de routerLink é followers
routerLink="followers"
Neste caso o endereço buscado será
host:porta/followers/followers
Caso utilize './', a engine se mantém no mesmo caminho, se utilizar '../', volta um nível na hierarquia.
routerLink='./'

Para possibilitar a busca pelo id do seguidor é preciso dizer ao routerLink qual é o id, para isso, precisamos utilizar o routerLink de forma dinâmica, como uma property binding, tendo em vista que os ids são dinâmicos, cada registro tem um id único, a sintaxe para isso é assim:
[routerLink]="['./', follower.id]"
O primeiro valor é o valor do caminho, o segundo valor é o parâmetro, no caso, o valor de id.
